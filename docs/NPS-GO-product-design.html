    • 1. 产品组件设计

    NPS GO 不是一个纯软件产品，而是一套 "模板 + 工具 + 指导"的组合系统。共包含 5 个核心组件：

    1.1 NPS 问卷模板库

    目标： 让客户 10 分钟内搭建出一份专业的 NPS 问卷，不需要任何专业知识。 设计原则：
    • 按行业分类，开箱即用
    • 问卷结构统一但内容行业化
    • 控制在 2 分钟内完成（移动端优先）
    行业模板清单（首发 4 个行业）：
    行业模板名称核心驱动维度
    餐饮NPS GO · 餐饮版菜品口味、食品安全、服务态度、环境氛围、性价比、上菜速度
    美业NPS GO · 美业版技师专业度、沟通理解、产品效果、卫生安全、服务细节、隐私保护
    潮牌/零售NPS GO · 潮牌版产品设计、限量稀缺感、社群文化、穿搭场景、线下体验
    小电商/私域NPS GO · 电商版商品描述一致性、物流速度、包装体验、售后响应、私域服务
    每个模板的标准结构： 
    ┌─ 开场语（30秒说明）

    ├─ 核心 NPS 题（0-10 分推荐度）
    

    ├─ 驱动因素深挖（5-8 题，行业定制）
    

    ├─ 开放题："如果我们能改进一点，您最希望是？"
    

    ├─ 用户标签（消费频次/客单价/获客渠道）
    

    └─ 致谢 + 激励（优惠券/积分/抽奖入口）
    交付物：
    • 可直接导入问卷平台的模板文件
    • 每题的出题逻辑说明（为什么问这道题）
    • 推荐发放时机说明（消费后多久发、通过什么渠道发）

    1.2 数据分析看板

    目标： 让客户看到数据后 5 秒内理解"我的客户满意度怎么样"。 看板层级（3 层）： 第一层：总览仪表盘
    • NPS 总分（-100 ~ +100），大数字展示
    • 推荐者/被动者/贬损者占比环形图
    • 与行业基准对比（如有）
    • 本月 vs 上月趋势折线图
    第二层：驱动因素分析
    • 各驱动维度的平均分雷达图
    • 贬损者集中在哪几个维度（红色高亮）
    • 推荐者最满意的维度（绿色高亮）
    • 改进优先级矩阵（重要性 × 满意度四象限图）
    第三层：客户明细
    • 每条问卷的原始回答
    • 客户标签筛选（按渠道/频次/客单价）
    • 可导出 Excel 做进一步分析
    实现方式：
    • 基础版： 飞书多维表格（自带仪表盘视图，零成本）
    • 进阶版： 飞书多维表格 + 定制仪表盘视图
    • 高级版： 独立 HTML 在线看板（可嵌入客户自己的管理系统）

    1.3 客户分群机制

    分群标准（基于经典 NPS 方法论）：
    分群NPS 评分定义标记方式
    🟢 推荐者 (Promoter)9-10忠诚的品牌拥护者，会主动推荐绿色标签
    🟡 被动者 (Passive)7-8满意但不热情，容易被竞品吸引黄色标签
    🔴 贬损者 (Detractor)0-6不满意的客户，可能传播负面口碑红色标签
    自动化分群流程： 
    客户提交问卷

    ↓
    

    系统自动读取 NPS 核心题分数
    

    ↓
    

    9-10 → 标记为「推荐者」→ 触发裂变激励任务
    

    7-8  → 标记为「被动者」→ 触发体验提升任务
    

    0-6  → 标记为「贬损者」→ 触发预警回访任务
    

    ↓
    

    同步到客户明细表 + 看板统计
    分群后的推荐行动：
    分群推荐行动
    推荐者发送感谢信 + 专属推荐奖励（如"推荐朋友各得 XX 元"）+ 邀请写好评/UGC
    被动者识别扣分维度 → 针对性改善（如"等位太久"则优化预约机制）
    贬损者24小时内专人回访 → 了解具体问题 → 制定挽回方案 → 闭环跟踪

    1.4 改进行动清单

    目标： 不只给数据，还要告诉客户"接下来该做什么"。 生成机制：模板化 + 数据驱动 第一部分：自动生成（基于数据）
    • 系统根据贬损者集中维度，自动生成 Top 3 改进方向
    • 每个方向附带：问题描述、影响客户数、建议行动
    示例输出： 
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    📋 NPS GO 改进行动清单（2026年6月）
    

    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    


    🔴 优先级 1：上菜速度
    

    问题：38% 的贬损者提到"等待时间过长"
    

    影响：本月 12 位贬损者中有 5 位因此打低分
    

    行动：
    

    □ 优化备餐流程，高峰期预切配
    

    □ 增加等位时的小食/饮品供应
    

    □ 设置"预计等待时间"告知机制
    

    预期：减少 30% 的等待相关投诉
    


    🟡 优先级 2：性价比感知
    

    问题：25% 的被动者认为"价格偏高"
    

    行动：
    

    □ 推出午市套餐/工作日特惠
    

    □ 增加菜品份量展示（如"2-3人份"标注）
    

    □ 收集"觉得值"的客户反馈，强化价值传达
    


    🟢 优先级 3：服务态度
    

    问题：15% 的贬损者提到服务冷淡
    

    行动：
    

    □ 制定服务 SOP（迎客/点餐/送客话术）
    

    □ 每月评选"服务之星"激励
    

    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    第二部分：行业最佳实践库（模板化）
    • 每个行业配套 10-15 条常见改进措施
    • 客户可从库中勾选，生成自己的行动清单
    • 提供执行难度和预期效果评估

    1.5 追踪复测机制

    目标： 让客户看到改进是否有效，形成"测量→改进→再测量"的闭环。 复测节奏建议：
    阶段时间动作
    基线测量第 1 周发放首次 NPS 问卷，建立基线数据
    改进执行第 2-3 周执行改进行动清单
    首次复测第 4 周发放第二次问卷，对比基线
    常规复测每月/每季度持续监测 NPS 变化趋势
    追踪看板功能：
    • NPS 分数趋势线（基线 → 复测 1 → 复测 2 → ...）
    • 各维度分数变化对比
    • 贬损者转化率（贬损者→被动者→推荐者的转化路径）
    • 改进行动完成度追踪（checklist 状态）
    交付物：
    • 复测问卷模板（与首次问卷保持核心题一致，驱动因素可微调）
    • 趋势对比报告模板
    • 复测提醒（通过飞书/微信提醒客户该复测了）

    3.2 各组件工具选型

    问卷工具

    工具优势劣势推荐场景成本
    飞书表单 ⭐首选与多维表格无缝打通；数据自动入库；零代码问卷逻辑跳转较弱飞书生态内的客户免费
    问卷星功能成熟；逻辑跳转强大；支持微信发放数据需要手动导出/对接需要复杂逻辑或微信发放免费版可用，专业版 ¥300/年
    腾讯问卷微信生态友好；免费额度大分析功能较弱微信私域为主的客户免费
    金数据设计感好；API 完善免费版限制多对问卷颜值有要求的客户基础版 ¥199/年
    推荐策略： 默认使用飞书表单（成本为零，数据自动流转），对需要微信发放的客户提供问卷星模板。

    数据分析工具

    工具优势劣势推荐场景成本
    飞书多维表格 ⭐首选仪表盘视图内置；自动汇总公式；协作方便复杂图表有限所有客户免费
    Excel/Google Sheets灵活度最高；客户熟悉无实时看板；协作不便客户偏好 Excel 的场景免费
    FineBI专业 BI；可视化强大学习成本高；价格贵数据量大的高级客户免费社区版
    推荐策略： 飞书多维表格作为默认方案。预置 NPS 计算公式、分群自动标记、仪表盘视图模板，客户只需录入数据即可看到看板。

    报告交付工具

    工具优势劣势推荐场景成本
    飞书文档 ⭐首选在线协作；可嵌入多维表格图表外部分享需设置权限日常报告免费
    HTML 在线看板交互性强；可定制品牌视觉需要开发和托管高级版客户开发成本
    PDF 报告正式感强；易于存档和分享静态；无法交互月度/季度正式报告生成工具免费
    推荐策略： 日常用飞书文档 + 多维表格看板（实时数据），月度/季度输出 PDF 正式报告。

    客户沟通工具

    工具推荐场景
    企业微信 ⭐与客户的日常沟通（群聊、文件传输）
    飞书内部团队协作
    微信小微客户直接对接

    3.3 推荐工具组合方案

    标准方案（零成本）： 
    飞书表单（问卷） → 飞书多维表格（数据+看板） → 飞书文档（报告）
    • 问卷数据自动流入多维表格
    • 多维表格内置仪表盘视图做可视化
    • 飞书文档输出月度报告
    • 总成本：¥0
    进阶方案（低成本）： 
    问卷星（问卷） → 飞书多维表格（数据+看板） → HTML 看板（在线报告）
    • 问卷星处理复杂问卷逻辑 + 微信发放
    • 数据通过 API 或手动导入多维表格
    • HTML 看板提供更好的可视化体验
    • 总成本：约 ¥300-1000/年

    4. 定价建议

    4.1 定价逻辑

    定价考量因素：
    • 目标客户是小微商家，价格敏感度高
    • 核心价值在于"方法论 + 模板 + 落地指导"，而非纯软件
    • 需要让客户觉得"比自己摸索划算"
    • 首发阶段以获取客户和口碑为主，不宜定价过高
    市场参考：
    • 问卷星专业版：¥300/年
    • 普通 SaaS CRM：¥200-500/月
    • 咨询公司 NPS 项目：¥50,000-200,000/项目
    • 本产品的定位：填补"免费工具"和"大型咨询项目"之间的空白

    5. 附录

    5.1 NPS 计算公式

    NPS = 推荐者占比(%) - 贬损者占比(%)


    示例：
    

    • 100 个客户参与评分
      • 

    • 60 人打 9-10 分（推荐者 = 60%）
      • 

    • 15 人打 0-6 分（贬损者 = 15%）
      • 

    • NPS = 60% - 15% = 45
      •
    NPS 分数解读：
    分数范围评价说明
    70+世界级极少数企业能达到
    50-69优秀客户忠诚度很高
    30-49良好有改进空间
    0-29一般需要关注核心问题
    < 0警告贬损者多于推荐者，急需改进
