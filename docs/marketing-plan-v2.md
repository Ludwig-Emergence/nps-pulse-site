## 1. 核心思路：产品驱动增长

### 1.1 增长飞轮

```
用户做诊断 → 拿到结果 → 结果值得截图/分享 → 朋友看到好奇 → 朋友也来测
     ↑                                                          |
     └──────────────────────────────────────────────────────────┘
```

**关键逻辑：** 不是"做推广"，是"让工具本身值得传播"。

增长动力来自产品本身，而不是运营投入：
- 诊断结果具有社交货币属性（"我的 NPS 是 72 分，你呢？"）
- 结果页面设计精美，天然适合截图分享
- 分享行为对用户有正向收益（展示专业度、引发讨论）

### 1.2 与 v1 的本质区别

| 维度 | v1（重运营） | v2（产品驱动） |
|------|-------------|---------------|
| 增长引擎 | 内容运营 + 投放 + 私域 | 产品本身 + 用户自发传播 |
| 人力需求 | 全职内容运营 1 名 | Lu 自己 + AI 辅助 |
| 核心资产 | 内容矩阵 + 社群 | 诊断工具 + 分享机制 |
| 成本结构 | ¥8,000-17,000/月 | ¥0（域名已购） |
| 可扩展性 | 线性增长（投入=产出） | 指数增长（用户越多传播越快） |

### 1.3 砍掉的内容及理由

| 砍掉项 | 理由 |
|--------|------|
| 小红书企业号蓝V | 个人号更灵活、零成本、MVP 阶段够用 |
| 全职内容运营人员 | Lu 自己 + AI 写内容，MVP 验证后再扩 |
| 薯条投放预算 | 自然流量验证内容质量，不花钱放大 |
| 公众号 | MVP 阶段不需要，官网+小红书足够承接 |
| 企业微信 | 用个人微信即可，降低启动门槛 |

## 3. 小红书个人号运营

### 3.1 账号设置

**账号名称：** NPS Pulse 或类似（个人号，不认证）

**简介模板：**
```
📊 专注客户满意度管理（NPS）
🎯 帮主理人/小商家用数据留住客户
🔗 做一次免费 NPS 诊断 👉 [官网链接]
```

**头像：** 品牌 Logo 图标版（Canva/AI 生成）

### 3.2 内容策略

**原则：** 少而精，每篇都带诊断工具入口。

**发布频率：** 每周 2-3 篇（Lu 自己 + AI 辅助写）

**内容类型：**

| 类型 | 占比 | 示例 |
|------|------|------|
| 干货教程 | 50% | 「3 步设计 NPS 问卷」「客户不复购？可能你没问对问题」 |
| 诊断结果展示 | 30% | 「这家咖啡店 NPS 只有 12 分，问题出在哪？」（引好奇→来做诊断） |
| 工具/模板 | 20% | 「NPS 计算模板，直接套用」 |

**关键：每篇笔记都要引导用户去官网做诊断。**

### 3.3 首批 5 篇选题

| # | 标题 | 目的 |
|---|------|------|
| 1 | 「你的客户满意度多少分？3 个问题快速自测」 | 引发好奇，引导做诊断 |
| 2 | 「NPS 是什么？主理人必懂的 1 个关键指标」 | 科普概念 |
| 3 | 「客户不复购？可能是你没问对这个问题」 | 痛点切入 |
| 4 | 「我用 3 个问题，把复购率从 15% 提到 38%」 | 案例+数据 |
| 5 | 「NPS 分数怎么算？1 分钟学会」 | 工具引流 |

### 3.4 冷启动策略

**不花钱的获客方式：**
1. 在小红书搜索相关关键词，给优质笔记真诚评论（不打广告，建立存在感）
2. 在目标社群（主理人/电商/创业群）分享 NPS 干货
3. 朋友圈发布 2-3 条相关内容（不刷屏）
4. 邀请 5-10 个目标用户免费体验诊断

## 5. 诊断工具的分享传播机制

这是整个增长飞轮的核心。工具本身要"值得传播"。

### 5.1 诊断结果页设计原则

**让结果具有社交货币属性：**
- 结果页设计精美，适合截图
- 分数可视化（仪表盘/进度条/评分卡）
- 包含一个简短的评价（"你的客户满意度处于中等偏上水平"）
- 显示"超过 XX% 的商家"（社交比较）
- 底部带品牌水印/二维码（自然传播品牌）

### 5.2 分享触发点

```
用户完成诊断
    ↓
看到结果页（设计精美、有洞察）
    ↓
自然反应："哇，我 NPS 是 65 分，还不错" → 截图
    ↓
分享到朋友圈/发给朋友/发到群里
    ↓
朋友看到："这是什么？我也想测" → 扫码/点链接
```

### 5.3 具体机制

| 机制 | 说明 |
|------|------|
| **结果截图分享** | 结果页设计为 9:16 竖图比例，天然适合手机截图 |
| **分享话术引导** | 结果页底部提示："分享给朋友，看看他们的 NPS 是多少" |
| **比较心理** | "你的分数超过了 62% 的商家" — 人都想知道自己的排名 |
| **结果页带链接** | 水印/二维码指向官网诊断入口，新用户可直接来测 |
| **社交互动** | "你的 NPS 是多少？评论区聊聊" — 小红书笔记引导互动 |

### 5.4 传播增长模型

```
Phase 1: Lu 自己 + 种子用户（10-20 人做诊断）
    ↓
Phase 2: 种子用户分享结果，带来更多用户（口碑传播）
    ↓
Phase 3: 小红书内容持续引流（内容+工具双引擎）
    ↓
Phase 4: 自然增长（工具本身成为获客渠道）
```
